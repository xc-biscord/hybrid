📘 Documentation de l’API Biscord

Référence des endpoints REST exposés sous /api/. En production : https://biscord-api-stg.xcsoftworks.com/api/. En local : http://127.0.0.1:8000/api/.

Authentification

L’authentification repose sur la session PHP (cookie PHPSESSID). On se connecte via login.php (ou le flux passkey), puis la session est réutilisée pour les appels suivants. Il n’existe pas encore de token d’API ni d’OAuth public.

Format des données

Sauf mention contraire, les corps de requête et les réponses sont en JSON. Convention générale :

{
  "success": true
}

En cas d’erreur :

{
  "success": false,
  "error": "Message lisible"
}

Codes HTTP courants : 200 succès, 400 requête invalide, 401 non authentifié, 403 interdit, 404 introuvable, 405 méthode non autorisée.

🔐 Authentification & session

login.php — POST

Connecte un utilisateur par identifiant (username ou e-mail) + mot de passe et ouvre la session.

{ "username": "alice", "password": "secret" }

Succès :

{ "success": true }

Erreur (identifiants invalides) : 401 { "success": false, "error": "Identifiants invalides" }

register.php — POST

Crée un compte (username, e-mail, mot de passe) puis le profil associé.

{ "username": "alice", "email": "alice@example.com", "password": "secret" }

Succès : { "success": true }. Conflit (username/e-mail déjà pris) : erreur success:false.

check_auth.php — GET

Indique si une session est active. Retourne l’identité courante le cas échéant.

{ "authenticated": true, "user_id": 1001 }

logout.php — GET

Détruit la session (session_unset + session_destroy) puis redirige (302) vers /index.html. Ne renvoie pas de corps JSON.

🔑 Passkeys / WebAuthn (PoC)

Authentification forte additive, sans toucher à login.php. Le login passkey se fait en deux temps : on récupère d’abord les méthodes disponibles, puis on déroule le challenge WebAuthn.

login_methods.php — POST

Retourne les méthodes de connexion disponibles pour un identifiant (sans révéler si le compte existe).

{ "identifier": "alice" }
{ "success": true, "password": true, "passkey": false }

passkey_login_options.php / passkey_login_verify.php — POST

passkey_login_options.php renvoie un challenge WebAuthn ; passkey_login_verify.php vérifie l’assertion signée par l’authentificateur et ouvre la session.

passkey_register_options.php / passkey_register_verify.php — POST

Enregistrement d’une passkey pour l’utilisateur connecté : génération des options de création, puis vérification et stockage de la clé publique (COSE) et du compteur anti-rejeu.

passkey_list.php (GET) / passkey_delete.php (POST)

Liste les passkeys de l’utilisateur connecté, ou en supprime une par id. La suppression d’une passkey appartenant à un autre utilisateur est refusée.

{ "id": 3001 }

👥 Utilisateurs & profils

get_all_users.php — GET

Liste des utilisateurs (réservé à l’administration globale). Session requise.

get_profile.php — GET

Profil de l’utilisateur connecté.

{ "success": true, "profile": { "bio": "...", "avatar_url": "...", "status": "disponible" } }

get_user_profile.php — GET

Profil public d’un utilisateur ciblé via ?user_id=<int>.

update_profile.php — POST

Met à jour le profil de l’utilisateur connecté (bio, avatar, statut). Charge utile vide acceptée.

{ "bio": "Hello", "status": "disponible" }

update_account.php — POST

Met à jour le compte (username, e-mail, mot de passe). Le changement de mot de passe exige le mot de passe actuel.

{ "current_password": "...", "new_password": "..." }

🧩 Serveurs

get_servers.php — GET

Serveurs dont l’utilisateur connecté est membre. Session requise (sinon 401).

create_server.php — POST

Crée un serveur (l’appelant en devient propriétaire).

{ "name": "Mon serveur" }

get_server_name.php — GET

Nom d’un serveur via ?id=<int>. Session requise.

get_user_servers.php — GET

Serveurs d’un utilisateur donné via ?user_id=<int> (usage administration). Paramètre manquant/invalide : 400.

📺 Salons (channels)

get_channels.php — GET

Salons d’un serveur via ?server_id=<int>. Session requise.

create_channel.php — POST

Crée un salon dans un serveur.

{ "server_id": 1101, "name": "general" }

💬 Messages publics

get_messages.php — GET

Historique d’un salon via ?channel_id=<int>. Session requise.

send_message.php — POST

Poste un message dans un salon.

{ "channel_id": 1201, "content": "Bonjour !" }

delete_message.php — POST

Supprime un message (auteur ou modérateur selon les droits).

{ "message_id": 1401 }

🔗 Invitations

create_invite.php — POST

Génère un code d’invitation pour un serveur (paramètre server_id en POST natif). Retourne le code créé.

accept_invite.php — POST

Permet à un utilisateur connecté de rejoindre un serveur via un code.

{ "code": "abc123" }

Succès :

{ "success": true, "server_id": 42 }

Erreurs : données manquantes ou { "success": false, "error": "Invitation invalide." }.

📨 Messages privés (DM)

start_dm.php — POST

Ouvre (ou retrouve) la conversation privée avec un autre utilisateur.

{ "user_id": 1002 }

get_dm_messages.php — GET

Messages d’une conversation via ?conversation_id=<int>. 401 non connecté, 403 non participant, 404 conversation inconnue.

send_dm.php — POST

Envoie un message privé dans une conversation.

{ "conversation_id": 2001, "content": "Salut !" }

get_dm_notifications.php — GET

Conversations comportant des messages non lus pour l’utilisateur connecté (calcul basé sur dm_reads).

🛡️ Modération & rôles

get_my_server_role.php — GET

Rôle de l’utilisateur connecté dans un serveur via ?server_id=<int> (P2, P3 ou member).

get_users_in_server.php — GET

Membres d’un serveur (avec leur rôle) via ?server_id=<int>.

set_member_role.php — POST

Modifie le rôle d’un membre (réservé aux administrateurs de serveur).

{ "server_id": 1101, "user_id": 1002, "role": "P3" }

kick_member.php — POST

Exclut un membre d’un serveur.

{ "server_id": 1101, "user_id": 1002 }

ban_user.php — POST

Bannit un utilisateur d’un serveur (modération).
